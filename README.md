# PARA Method — Obsidian Vault Template

An [Obsidian](https://obsidian.md) vault template for organizing your personal
notes with **PARA**, the method created by [Tiago Forte](https://fortelabs.com/blog/para/).

PARA sorts everything you keep into four categories by **actionability** rather
than by topic — so notes live where you'll actually use them, not in a tangle of
subjects you have to maintain.

## The four categories

| Folder | What goes here |
| --- | --- |
| `1_projects` | Short-term efforts with a goal and a deadline — things you're actively working to finish. |
| `2_areas` | Ongoing responsibilities to maintain over time (health, finances, work role). No end date. |
| `3_resources` | Topics and interests you want to reference later, but that aren't tied to a current project. |
| `4_archives` | Inactive items from the other three — completed projects, dormant areas, stale resources. |

A fifth folder, `0_inbox`, is a capture spot for anything not yet sorted. Process
it regularly and move each note into the category that fits.

## How it works

- **Capture** new notes in `0_inbox`.
- **Sort** them into Projects, Areas, or Resources based on how actionable they are.
- **Archive** anything that goes inactive — don't delete it, just move it to
  `4_archives` so it's out of the way but recoverable.
- **Review** periodically: finished projects and stale notes get archived; archived
  items can come back when they become relevant again.

The same note can be moved freely between folders as its role changes — a resource
can become a project, a project can be archived, and so on.

## Getting started

1. Open this folder as a vault in Obsidian (*Open folder as vault*).
2. Read `PARA Method.md` for a fuller explanation of the system.
3. Start dropping notes into `0_inbox` and sorting from there.
